Современная распределенная платформа для командной работы и управления задачами, 
построенная на микросервисной архитектуре по принципу канбан-досок (аналог Trello/Jira).

Система поддерживает совместные воркспейсы, гибкое разграничение ролей, динамическое перемещение задач, 
сквозной аудит действий и асинхронную систему уведомлений через брокер сообщений.

---

## 🏗️ Микросервисная архитектура

Система разделена на три изолированных сервиса с асинхронным взаимодействием:

1. **Backend Service (`task-tracker-backend`)**<p>
   Центральное ядро системы. Управляет бизнес-логикой авторизации, досок, секций, меток и задач. 
   Обеспечивает ролевую модель доступа (RBAC), генерирует события (Events) для Kafka и логирует активность пользователей.

2. **Scheduler Service (`task-tracker-scheduler`)**<p>
   Сервис-планировщик на базе Spring Scheduler. 
   Пакетно обрабатывает дедлайны задач в фоновом режиме и отправляет триггеры уведомлений в брокер сообщений.

3. **Email Sender Service (`task-tracker-email-sender`)**<p>
   Изолированный обработчик уведомлений (Consumer). Слушает топики Kafka, гарантируя асинхронную и отказоустойчивую 
   отправку писем через SMTP-протокол без замедления работы основного API.

## ✨ Ключевой функционал Backend-сервиса

- 🔐 **Advanced Security:** 
   Безопасная stateless-аутентификация с использованием парных JWT-токенов. 
   Срочный Access Token передается в теле ответа, а долгоживущий Refresh Token — в защищенной HttpOnly куке.
- 👥 **Collaborative Workspaces (Boards):**
   Создание изолированных рабочих пространств. Владелец доски может приглашать других пользователей системы и 
   гибко управлять их ролями (Owner, Editor, Viewer).
- 📊 **Dynamic Board Structure:** 
   Асинхронная отправка почты через Kafka, что исключает задержки API при регистрации.
- 🔄 **Task Lifecycle & Drag-and-Drop:**
   Полноценное управление задачами — быстрое переключение готовности (Toggle), 
   привязка/отвязка цветовых меток (Labels) и изменение приоритетов/колонок (Move/Drag-and-Drop).
- 🔍 **Advanced Search & Filtering:**
   Гибкий поиск задач по сложным составным критериям (фильтрация по авторам, исполнителям, меткам и статусам) 
   с серверной пагинацией и сортировкой.
- 📜 **Audit Trail (Activity Logs):**
   Система автоматически фиксирует хронологию действий. Доступны логи как личной активности пользователя, 
   так и общие ленты изменений в рамках конкретной доски.
- 👤 **User Profiles & Avatars:** Управление профилем пользователя, 
включая загрузку и смену аватарок через MinIO S3-хранилище.
- 📅 **Task Enhancements (Deadlines & Assignees):** Расширенная карточка задачи: 
установка дедлайнов для контроля сроков и назначение ответственных исполнителей.
- 📎 **File Attachments:** Поддержка прикрепления любых файлов к задачам. Файлы безопасно хранятся в S3-объектах.
- 💬 **Collaboration (Comments):** Система комментариев для обсуждения задач внутри команды.

## 🛠️ Стек технологий

### 💻 Backend

- ☕ **Java 21** (LTS)
- 🍃 **Spring Boot 4.0.0** (Web, Security, Data JPA, Scheduler, Mail, Validation)
- 🐘 **PostgreSQL** — основная реляционная база данных
- 🌊 **Liquibase** — управление миграциями схемы БД
- 🧠 **Redis** — хранение и инвалидация токенов
- 🔐 **JJWT (Java JWT)** — реализация механизма аутентификации через токены
- 📤 **Apache Kafka** — распределенный брокер сообщений для асинхронных задач
- 📦 **MinIO** — S3-совместимое объектное хранилище для пользовательских аватарок и вложений к задачам.
- 🗺️ **MapStruct** — высокопроизводительный маппинг DTO ↔ Entity
- 🌶️ **Lombok** — минимизация шаблонного кода (Boilerplate)
- 📖 **Springdoc OpenAPI (Swagger UI)** — интерактивная документация API

### 🏗️ DevOps & Testing

- 🐳 **Docker & Docker Compose** — контейнеризация приложения и инфраструктуры
- 🧪 **JUnit 5 & Mockito** — unit-тестирование логики
- 🧰 **Testcontainers** — запуск реальных сервисов (Postgres, Kafka, Redis) в Docker для интеграционных тестов
- 🛠️ **Spring Boot Test** — комплексное тестирование контекста приложения

```

git clone https://github.com/Zoxid-Iskandarov/task-tracker.git
```

## 📚 Спецификация REST API

Вся интерактивная документация к эндпоинтам backend-сервиса полностью автоматизирована и вынесена 
в чистые Java-интерфейсы. Вы можете выполнять тестовые запросы, смотреть схемы входящих DTO 
и ответы напрямую через визуальный интерфейс.

После запуска проекта спецификация во всех подробностях доступна по адресу:

👉 http://localhost:8080/swagger-ui/index.html

**Архитектурные модули API (Tags):**

**Authentication** — Регистрация, авторизация (Sign-In/Sign-Up), 
обновление сессий через защищенные куки (Refresh) и безопасный Sign-Out.

**User Profile** — Получение метаданных профиля текущего пользователя и просмотр персонального лога активности.

**Boards Management** — Управление рабочими пространствами, поиск пользователей для инвайтов, 
менеджмент ролей участников, просмотр логов изменений доски.

**Sections Management** — Операции над колонками внутри досок (создание, переименование, удаление) и 
получение упорядоченного списка задач в них.

**Labels Management** — Создание уникальных цветовых тегов/меток для кастомизации и группировки задач.

**Tasks Management** — Полный жизненный цикл задачи: просмотр, назначение исполнителей, дедлайнов, перемещение между 
секциями (Drag-and-Drop), привязка меток и управление статусом выполнения.

**Comments Management** — Инструменты для обсуждения задач в реальном времени.

**Attachments Management** — Загрузка файлов, получение защищенных ссылок для скачивания и менеджмент вложений.
